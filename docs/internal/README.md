# Internal Notes

This directory is the home for internal, human-written operational material that does not belong at the repository root.

Use it for:

- handoff drafts
- stakeholder message drafts
- operator checklists
- internal rollout notes

Suggested structure:

- `communications/`: emails, handoffs, stakeholder drafts
- `ops/`: runbooks, checklists, temporary operating notes that should be retained

Do not store secrets here. Deployment secrets belong in local env files that stay untracked.
